```plantuml
@startuml
' Feature: Periodic Score Update (UC-H3)
' As the Highscore Display System, I want to update the data periodically
' So that the displayed information remains current

' Define participants in the system
participant "Update Timer" as Timer
box "Highscore Display System"
    participant "System Core" as System
    participant "DOM/UI Elements" as UI
end box
participant Backend

' Styling for better readability and theme
skinparam handwritten false
skinparam defaultFontName "Cascadia Code"
skinparam sequence {
    ArrowColor #6A0DAD
    ActorBorderColor #6A0DAD
    LifeLineBorderColor #6A0DAD
    LifeLineBackgroundColor #F0E6FA
    ParticipantBorderColor #6A0DAD
    ParticipantBackgroundColor #F0E6FA
    ParticipantFontColor #333333
    StereotypeFontColor #6A0DAD
    BoxBorderColor #6A0DAD
    BoxBackgroundColor #F0E6FA
    NoteBorderColor #6A0DAD
    NoteBackgroundColor #FDFDFD
    NoteFontColor #333333
}

' Basic Flow: Sessiz Güncelleme (Silent Update)
' Scenario: Updating scores without visual disruption
' Alternatif Akış A1: Güncelleme Başarısız
' Scenario: Handling update failure gracefully

Timer -> System: triggerUpdate(interval: 30s)
activate System

System -> Backend: fetchScores()
activate Backend

alt Successful Data Fetch (Basic Flow)
    Backend --> System: returnScores(latestData)
    deactivate Backend
    System -> UI: updateElements(latestData)
    note right of UI: Update happens "silently"\n(no page reload or blink)
else Network Request Fails (Alternatif Akış A1)
    Backend --x System: requestFailed()
    deactivate Backend
    note left of System: Keep displaying existing data
    System -> UI: displayWarningIcon("Data stale / Update failed")
    note over System: System will retry at the next scheduled interval\n(upon next Timer trigger)
end alt

deactivate System

@enduml
```
