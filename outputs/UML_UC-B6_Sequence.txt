```plantuml
@startuml
' Actors and Systems involved in the session initiation process
participant KioskClient
participant BackendSystem as "Backend System"
participant WalletService as "Wallet Service"
participant FinancialLoggingService as "Financial Logging Service"

' Main flow: Successful Session Authorization
KioskClient -> BackendSystem: Start Session Request (Card ID, Game ID)
activate BackendSystem

BackendSystem -> WalletService: Check Balance (Card ID, Game Cost)
activate WalletService

alt Sufficient Balance (Balance > Game Cost)
    WalletService --> BackendSystem: Balance Check Result (Sufficient)
    deactivate WalletService

    BackendSystem -> WalletService: Deduct Cost (Card ID, Game Cost)
    activate WalletService
    WalletService --> BackendSystem: Cost Deduction Confirmation
    deactivate WalletService

    BackendSystem -> FinancialLoggingService: Log Transaction (Type: Debit, Card ID, Game ID, Amount)
    activate FinancialLoggingService
    FinancialLoggingService --> BackendSystem: Log Confirmation
    deactivate FinancialLoggingService

    BackendSystem --> KioskClient: Session Authorized Response
else Insufficient Balance (Balance < Game Cost)
    ' Alternative Flow A1: Insufficient Balance
    WalletService --> BackendSystem: Balance Check Result (Insufficient)
    deactivate WalletService

    BackendSystem --> KioskClient: Insufficient Funds Error
    ' Note: As per scenario, wallet balance remains unchanged in this flow.
end

deactivate BackendSystem

' Exception Flow I2: Latency / Timeout Requirement
note right of BackendSystem
  **Real-time Response Requirement:**
  The Backend must send a response within "1 second".
  If processing exceeds this threshold, a timeout error
  should be logged internally by the Backend System.
end

@enduml
```
