@startuml
title Scenario: Dashboard successfully retrieves report data

actor Admin
participant "Dashboard UI" as UI
participant "Dashboard System" as DS
participant "Backend API" as BE
database "Data Store" as DB

Admin -> UI: Triggers "Fetch Financial Report"
activate UI
UI -> DS: Request financial report data
activate DS
DS -> BE: GET /reports/financial (Authorization: Bearer <token>)
activate BE
BE -> BE: Validate authorization token
note right of BE: Background: Secure HTTPS connection assumed
alt Token is Valid
    BE -> DB: Query financial data
    activate DB
    DB --> BE: Financial data
    deactivate DB
    BE --> DS: 200 OK, { "reportData": [...] } (JSON)
else Token is Invalid / Expired (handled by Scenario I2)
    BE --> DS: 401 Unauthorized (See Scenario I2)
end
deactivate BE
DS --> UI: Report data received
deactivate DS
UI -> Admin: Renders financial report data
deactivate UI

@enduml

@startuml
title Scenario: Retrieving configuration from Cache

actor Admin
participant "Dashboard UI" as UI
participant "Dashboard System" as DS
participant "Backend API" as BE
component Cache
database "Data Store" as DB

Admin -> UI: (Implicit) Dashboard loads, needs config
activate UI
UI -> DS: Requests static configuration data
activate DS
DS -> BE: GET /config/static
activate BE
note right of BE: Backend checks for cached data first
alt Configuration found in Cache
    BE -> Cache: Retrieve static configuration
    activate Cache
    Cache --> BE: Static configuration data
    deactivate Cache
    note over BE, Cache: Response time significantly lower than a database query
    BE --> DS: 200 OK, { "config": {...} } (JSON)
else Configuration not in Cache
    BE -> DB: Query static configuration
    activate DB
    DB --> BE: Static configuration data
    deactivate DB
    BE -> Cache: Store configuration in cache for future requests
    BE --> DS: 200 OK, { "config": {...} } (JSON)
end
deactivate BE
DS --> UI: Static configuration received
deactivate DS
UI -> Admin: (Implicit) Dashboard uses configuration for display
deactivate UI

@enduml

@startuml
title Scenario: Backend fails to respond within time limit

actor Admin
participant "Dashboard UI" as UI
participant "Dashboard System" as DS
participant "Backend API" as BE

Admin -> UI: User performs action requiring data
activate UI
UI -> DS: Sends request
activate DS
group "Request with 2-second Timeout"
    DS -> BE: Request data (timeout set to 2s)
    activate BE
    DS o-> DS: Start 2-second timer
    ... 2 seconds pass ...
    note left of DS: Timer expires before response from Backend
    DS x-> BE: Request cancelled
    deactivate BE
    DS -> UI: Signal "System not responding" error
end
deactivate DS
UI -> Admin: Display "System not responding" error message
deactivate UI

@enduml

@startuml
title Scenario: Handling unauthorized API responses

actor Admin
participant "Dashboard UI" as UI
participant "Dashboard System" as DS
participant "Backend API" as BE
participant "Login Screen" as Login

Admin -> UI: User tries to access data
activate UI
UI -> DS: Sends request (with invalid or expired token)
activate DS
DS -> BE: GET /data (Authorization: Bearer <invalid_token>)
activate BE
BE -> BE: Validate Authorization Token
note right of BE: Token is invalid or expired
BE --> DS: 401 Unauthorized
deactivate BE
DS -> UI: Receives 401 Unauthorized status
deactivate DS
UI -> Login: Redirect user to login screen
activate Login
Login -> Admin: Display login prompt for re-authentication
deactivate Login
deactivate UI

@enduml
