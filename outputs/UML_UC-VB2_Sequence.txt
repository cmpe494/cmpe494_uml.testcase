```plantuml
@startuml
title Scenario: Seamless switch to offline mode

participant User
participant Application
participant NetworkMonitor
participant DataManager
participant LocalCache
participant SyncQueue

User -> Application: Use Application (previously cached essential data)
note over Application: Application is initially online

NetworkMonitor -> NetworkMonitor: Detect internet connection lost
activate NetworkMonitor
NetworkMonitor -> Application: Notify: "Connection Lost"
deactivate NetworkMonitor

activate Application
Application -> Application: Switch to "Offline Mode"
Application -> DataManager: Set data source preference: LocalCache
deactivate Application

User -> Application: Request Data (e.g., View content)
activate Application
Application -> DataManager: Request Data
activate DataManager
DataManager -> LocalCache: Retrieve Data
activate LocalCache
LocalCache --> DataManager: Serve Data
deactivate LocalCache
DataManager --> Application: Data from Cache
deactivate DataManager
Application --> User: Display Data
deactivate Application

User -> Application: Perform Write Operation (e.g., Save draft)
activate Application
Application -> DataManager: Submit Write Operation
activate DataManager
DataManager -> SyncQueue: Enqueue Write Operation (for later synchronization)
activate SyncQueue
SyncQueue --> DataManager: Queued
deactivate SyncQueue
DataManager --> Application: Operation Queued
deactivate DataManager
Application --> User: Write Operation processed (locally)
deactivate Application
@enduml

@startuml
title Scenario: Attempting online-only actions while offline

participant User
participant Application
participant OfflineModeManager
participant OnlineContentService

note over Application: System is in "Offline Mode"

User -> Application: Attempt "Download New Content"
activate Application
Application -> OfflineModeManager: Query: Can perform "Download New Content"?
activate OfflineModeManager
alt Action requires internet connection
    OfflineModeManager -> OfflineModeManager: Check if internet is available
    OfflineModeManager --> Application: Block: "Internet connection required"
    deactivate OfflineModeManager
    Application -> Application: Block Request
    Application --> User: Display "Internet connection required" warning
else
    ' This path is not taken in this scenario
    OfflineModeManager --> Application: Allow
end
deactivate Application
@enduml

@startuml
title Scenario: Sync failure after reconnection

participant User
participant Application
participant NetworkMonitor
participant SyncManager
participant OnlineService
participant SyncQueue

note over Application: Application was in "Offline Mode" with queued data

NetworkMonitor -> NetworkMonitor: Detect internet connection restored
activate NetworkMonitor
NetworkMonitor -> Application: Notify: "Connection Restored"
deactivate NetworkMonitor

activate Application
Application -> SyncManager: Initiate Sync Offline Data
activate SyncManager
SyncManager -> OnlineService: Attempt to Sync Data (from SyncQueue)
activate OnlineService
alt Synchronization process encounters an error
    OnlineService --> SyncManager: Sync Error Response
    deactivate OnlineService
    SyncManager --> Application: Report Sync Failure
    deactivate SyncManager
    Application -> Application: Handle Sync Failure
    Application --> User: Display "Sync failed, please retry later" message
    Application -> SyncQueue: Keep data in local queue (do not clear)
    activate SyncQueue
    SyncQueue --> Application: Data retained
    deactivate SyncQueue
else
    ' This path is not taken in this scenario
    OnlineService --> SyncManager: Sync Success Response
end
deactivate Application
@enduml
```
