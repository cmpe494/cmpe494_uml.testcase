```plantuml
@startuml
title Screen Layout and Theme Management (UC-H6) - Visual Settings Update

participant Admin
participant "Display Settings Panel" as UI
participant "Configuration Service" as Backend
participant "Display Client"

Admin -> UI: Access "Display Settings" panel

alt Basic Flow: Applying new visual settings
    Admin -> UI: Update "<setting>" to "<value>"
    Admin -> UI: Click "Save"
    UI -> Backend: Request to Save Configuration (setting, value)
    activate Backend
    Backend -> Backend: Validate and Persist New Configuration
    Backend -> "Display Client": Push New Configuration
    activate "Display Client"
    "Display Client" -> "Display Client": Parse and Apply Changes (Theme, Font, Layout)
    note right of "Display Client": Changes applied immediately without restart
    "Display Client" --> Backend: Configuration Applied Successfully
    deactivate "Display Client"
    Backend --> UI: Configuration Saved Confirmation
    deactivate Backend
    UI --> Admin: Display "Settings Saved" confirmation
else Alternatif Akış A1: Display client fails to apply settings
    Admin -> UI: Send new configuration (implicitly via "Save" action)
    UI -> Backend: Request to Save Configuration (new config data)
    activate Backend
    Backend -> Backend: Validate and Persist New Configuration (if valid at backend)
    Backend -> "Display Client": Push New Configuration
    activate "Display Client"
    "Display Client" -> "Display Client": Attempt to Parse and Apply Settings
    ref over "Display Client": Cannot parse the settings (e.g., malformed data)
    "Display Client" -> "Display Client": Revert to previous working theme
    "Display Client" --> Backend: Report "Configuration Failed"
    deactivate "Display Client"
    Backend --> UI: Notify "Configuration Failed"
    deactivate Backend
    UI --> Admin: Display "Configuration Failed" to Admin
end

@enduml
```
