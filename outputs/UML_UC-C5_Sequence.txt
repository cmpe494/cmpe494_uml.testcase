```plantuml
@startuml
' Actors and Participants
actor User
participant KioskClient
participant InternetConnection
participant LocalBackend
database LocalDatabase

' Overall context for the feature
note over KioskClient, LocalDatabase: Feature: Offline Mode Communication (UC-C5)\nAs a Client Device (Kiosk/Game)\nI want to communicate with the Local Backend when internet is down\nSo that I can continue serving customers without interruption

' Background: global internet connection is disconnected, Local Backend is available
' This sets up the initial state for the scenarios.
InternetConnection -[hidden]-> KioskClient
note over InternetConnection, KioskClient: Given the global internet connection is disconnected
note over LocalBackend, LocalDatabase: And a Local Backend is available on the LAN

' Scenario: Switch to Local Backend for service
group Basic Flow: Switch to Local Backend for service
    KioskClient -> KioskClient: Detects internet loss
    activate KioskClient
    KioskClient -> KioskClient: Automatically switches to "Offline Mode"
    note right: KioskClient updates its internal routing logic

    User -> KioskClient: Requests data (e.g., product info)
    KioskClient -> LocalBackend: Routes data request to "Local Backend IP"
    activate LocalBackend
    LocalBackend -> LocalDatabase: Requests data from local database
    activate LocalDatabase
    LocalDatabase --> LocalBackend: Provides data
    deactivate LocalDatabase
    LocalBackend --> KioskClient: Provides data
    deactivate LocalBackend
    KioskClient --> User: Service continues seamlessly (data displayed)
    deactivate KioskClient
end

' Scenario: LAN failure during offline mode
group Exception Flow I1: LAN failure during offline mode
    note over KioskClient, LocalDatabase: Given the system is in "Offline Mode" (pre-condition from previous scenario)

    KioskClient x-- LocalBackend: Connection to Local Backend is lost
    activate KioskClient
    KioskClient -> User: Displays "Service Unavailable - Network Error" screen
    KioskClient -> KioskClient: Stops accepting user requests
    deactivate KioskClient
end

' Scenario: Requesting unsupported features in offline mode
group Exception Flow I2: Requesting unsupported features in offline mode
    note over KioskClient, LocalDatabase: System is in "Offline Mode" (pre-condition)

    User -> KioskClient: Requests "Global Leaderboard" service
    activate KioskClient
    KioskClient -> KioskClient: Checks if "Global Leaderboard" is supported in current mode
    alt KioskClient is in "Offline Mode"
        KioskClient -> KioskClient: Denies the request
        KioskClient -> User: Displays "Feature not available offline" message
    else KioskClient is in "Online Mode" (not this scenario)
        ' KioskClient -> GlobalBackend: Requests "Global Leaderboard" service
        ' GlobalBackend --> KioskClient: Provides data
        ' KioskClient --> User: Displays Global Leaderboard
    end
    deactivate KioskClient
end
@enduml
```
