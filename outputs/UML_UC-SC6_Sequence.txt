```plantuml
@startuml
participant Admin
participant VersionControlUI
participant VersionTrackingService
participant DeviceAgent as "Device Agent (on device)"
participant VersionDatabase
participant LoggingService

title Software Version Tracking - Component Interaction Flow

group Background Data Collection and Processing
    note over VersionTrackingService: This group represents the continuous or periodic processes\nthat gather and process version data from devices,\nhandling various scenarios before an Admin views them.

    loop For each registered device (e.g., Kiosk-01, Kiosk-02, Kiosk-Old)
        DeviceAgent -> VersionTrackingService: Report current software versions
        activate DeviceAgent
        activate VersionTrackingService

        alt Scenario: Handling devices with unreadable version info ("Kiosk-Old")
            DeviceAgent --> VersionTrackingService: Error: Cannot read version registry
            deactivate DeviceAgent
            VersionTrackingService -> VersionDatabase: Store device status: "Unknown" version, "Flag: Manual Inspection" (for Kiosk-Old)
            activate VersionDatabase
            VersionDatabase --> VersionTrackingService: Acknowledged
            deactivate VersionDatabase
        else Scenario: Detecting version inconsistency / Conflict
            DeviceAgent --> VersionTrackingService: Report version 1 (e.g., v2.1.0, timestamp A)
            DeviceAgent --> VersionTrackingService: Report version 2 (e.g., v2.0.9, timestamp B > A)
            deactivate DeviceAgent
            VersionTrackingService -> LoggingService: Log "Version Conflict Warning" (Device: Kiosk-Conflict, Conflicting Versions: v2.1.0, v2.0.9)
            activate LoggingService
            LoggingService --> VersionTrackingService: Log Acknowledged
            deactivate LoggingService
            VersionTrackingService -> VersionDatabase: Store/Update device version: Most recently received valid version (v2.0.9)
            activate VersionDatabase
            VersionDatabase --> VersionTrackingService: Acknowledged
            deactivate VersionDatabase
        else Scenario: Valid and Consistent Version Info ("Kiosk-01", "Kiosk-02")
            DeviceAgent --> VersionTrackingService: Report versions (e.g., Kiosk-01: Win10, v2.1.0)
            deactivate DeviceAgent
            VersionTrackingService -> VersionDatabase: Store/Update device version (Kiosk-01, Win10, v2.1.0, Status: Up-to-date)
            VersionTrackingService -> VersionDatabase: Store/Update device version (Kiosk-02, Win10, v2.0.9, Status: Outdated)
            activate VersionDatabase
            VersionDatabase --> VersionTrackingService: Acknowledged
            deactivate VersionDatabase
        end
        deactivate VersionTrackingService
    end
    note over VersionTrackingService, VersionDatabase: At this point, the database holds processed and aggregated\nversion data for all devices, including compliance statuses.
end group

group Basic Flow: Displaying version inventory
    Admin -> VersionControlUI: Navigates to "Version Control"
    activate VersionControlUI
    VersionControlUI -> VersionTrackingService: Request all device versions for display
    activate VersionTrackingService
    VersionTrackingService -> VersionDatabase: Query aggregated device version data
    activate VersionDatabase
    VersionDatabase --> VersionTrackingService: List of DeviceVersionSummary[]
    deactivate VersionDatabase
    VersionTrackingService --> VersionControlUI: Send processed device version inventory
    deactivate VersionTrackingService
    VersionControlUI --> Admin: Display version inventory table based on processed data:\n  | Device Name | OS Version | App Version | Status            |\n  | Kiosk-01    | Win10      | v2.1.0      | Up-to-date        |\n  | Kiosk-02    | Win10      | v2.0.9      | Outdated          |\n  | Kiosk-Old   |            | Unknown     | Manual Inspection |
    deactivate VersionControlUI
end group

@enduml
```
