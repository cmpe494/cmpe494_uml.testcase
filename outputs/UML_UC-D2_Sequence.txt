```plantuml
@startuml
title Dashboard Viewing and Monitoring (UC-D2)

actor "Manager/Admin" as User
participant DashboardUI as Dashboard
participant DashboardController as System
participant BackendDataService
participant LocalDataCache as Cache

activate User
User -> Dashboard: Navigates to Dashboard (User logged in with viewing permissions)
deactivate User

activate Dashboard
Dashboard -> System: Request initial metrics data
deactivate Dashboard

activate System
alt Basic Flow: Dashboard loads with real-time data
    System -> BackendDataService: Request real-time operational metrics
    activate BackendDataService
    BackendDataService --> System: Respond with Real-time Metrics Data
    deactivate BackendDataService
    
    System -> Cache: Store latest metrics for "Last Known" values
    activate Cache
    Cache --> System: Acknowledge storage
    deactivate Cache

    System -> Dashboard: Display Metrics: Zone Occupancy (Heatmap), Daily Revenue (Line Chart), Active Alerts (List View)
    activate Dashboard
    Dashboard -> System: Acknowledge display / Configure auto-refresh
    deactivate Dashboard
    System -> System: Auto-refresh scheduled (every 30 seconds)
    note right of System: This interaction repeats every 30 seconds, requesting fresh data.
else Alternatif Akış A1: Handling backend data failure
    System -> BackendDataService: Request real-time operational metrics
    activate BackendDataService
    BackendDataService --[X System: Fails to respond (timeout/error)
    deactivate BackendDataService
    
    System -> Dashboard: Display "Data unavailable" warning
    activate Dashboard
    deactivate Dashboard

    System -> Cache: Request "Last Known" cached values
    activate Cache
    Cache --> System: Respond with cached metrics data
    deactivate Cache

    System -> Dashboard: Display "Last Known" cached values (instead of a blank screen)
    activate Dashboard
    deactivate Dashboard
end
deactivate System

@enduml
```
